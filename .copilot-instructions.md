# Copilot Instructions – React Application

These instructions guide GitHub Copilot when generating code for this React project.

---

# Tech Stack

* React (latest)
* Functional Components only
* Hooks based architecture
* TypeScript preferred when possible
* TailwindCSS for styling
* Axios or Fetch for API calls
* Modular component architecture

---

# Project Structure

Follow this folder structure when generating new code.

```
src/
  components/
  pages/
  hooks/
  services/
  utils/
  types/
  assets/
  context/
  routes/
```

Example:

```
src/
  components/
    Button/
      Button.tsx
      Button.test.tsx
      Button.types.ts
  pages/
    Dashboard/
      Dashboard.tsx
  services/
    api.ts
```

---

# Component Guidelines

* Use **functional components**
* Prefer **named exports**
* Keep components small and reusable
* Separate UI and logic where possible
* Use hooks for state and side effects

Example component:

```tsx
import React from "react";

type ButtonProps = {
  label: string;
  onClick?: () => void;
};

export const Button: React.FC<ButtonProps> = ({ label, onClick }) => {
  return (
    <button
      className="px-4 py-2 rounded bg-blue-500 text-white"
      onClick={onClick}
    >
      {label}
    </button>
  );
};
```

---

# State Management

Prefer this order:

1. React Hooks (`useState`, `useReducer`)
2. Context API
3. Redux (only when necessary)

Example:

```tsx
const [count, setCount] = useState(0);
```

---

# API Layer

API logic must be separated into a **services folder**.

Example:

```
src/services/userService.ts
```

Example implementation:

```ts
import axios from "axios";

export const getUsers = async () => {
  const response = await axios.get("/api/users");
  return response.data;
};
```

Components should call services instead of making direct API calls.

---

# Custom Hooks

Business logic should be extracted into hooks.

Example:

```
src/hooks/useUsers.ts
```

```ts
import { useEffect, useState } from "react";
import { getUsers } from "../services/userService";

export const useUsers = () => {
  const [users, setUsers] = useState([]);

  useEffect(() => {
    getUsers().then(setUsers);
  }, []);

  return users;
};
```

# Styling

Use TailwindCSS.

Example:

```tsx
<div className="flex items-center justify-between p-4 bg-gray-100 rounded-lg">
  Dashboard
</div>
```

Avoid inline styles unless absolutely necessary.

---

# Error Handling

Always include error handling in async logic.

Example:

```ts
try {
  const data = await getUsers();
} catch (error) {
  console.error("Failed to fetch users", error);
}
```

# Code Generation Rules

When Copilot generates code:

* Always include imports
* Use TypeScript types when possible
* Keep components under ~150 lines
* Prefer reusable components
* Avoid large monolithic components
* Follow the defined folder structure
* Provide clear naming conventions

# Performance Guidelines

```

useCallback
```

```tsx
const memoizedValue = useMemo(() => computeExpensiveValue(data), [data]);
```
